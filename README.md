# Трекер выработки — учёт часов и зарплаты

Локальное приложение для учёта рабочих часов (от выработки) и расчёта зарплаты по месяцам.
Работает офлайн, данные хранятся на устройстве (IndexedDB). Устанавливается как приложение (PWA),
обновляется по сети, синхронизируется между ПК и телефоном.

**Версия:** 1.0.0-beta

## Запуск
Просто открой `index.html` в браузере (Chrome/Edge). Всё содержится в этой папке, сервер не нужен.

## Возможности
- 📅 Календарь: тип дня (рабочий, выходной, праздник, отпуск, больничный, отгул, прогул), часы и минуты, заметки
- 💰 Зарплата: расчёт по месяцам (наработанные часы × ставка), разбивка по типам, год в столбиках
- 📊 Статистика: графики заработка, распределение дней, часы по дням недели
- 🏠 Дашборд: сводка месяца, «сегодня», последние 7 дней, недавние записи
- ⚙️ Настройки: ставка, оплата больничного/отпуска, выходные, бэкап (экспорт/импорт)
- ☁️ Синхронизация между устройствами (Supabase)
- 🔄 Обновления по сети без переустановки (Service Worker)

## Структура
```
index.html            — точка входа
manifest.webmanifest  — манифест PWA
sw.js                 — service worker (офлайн + обновления)
css/styles.css        — стили и тема
icons/icon.svg        — иконка
js/
  version.js   — версия приложения (менять здесь)
  config.js    — типы дней, настройки по умолчанию
  utils.js     — даты, форматирование
  db.js        — IndexedDB (хранилище)
  salary.js    — расчёт зарплаты
  calendar.js  — календарь и редактор дня
  salaryview.js— страница «Зарплата»
  charts.js    — SVG-графики
  statsview.js — страница «Статистика»
  settings.js  — страница «Настройки»
  sync.js      — облачная синхронизация (Supabase)
  pwa.js       — регистрация SW и обновления
  background.js— анимированный фон
  app.js       — навигация и сборка всего вместе
```

## Инструкции по развёртыванию
- `DEPLOY-GITHUB.md` — публикация онлайн (GitHub Pages)
- `SYNC-SETUP.md` — настройка синхронизации (Supabase)
- `BUILD-EXE-APK.md` — сборка настоящих exe и apk (PWABuilder)

## Как выпускать обновления
1. Изменить файлы.
2. Поднять версию в `js/version.js` и `CACHE_VERSION` в `sw.js`.
3. Залить на хостинг. Клиенты сами предложат «Обновить».
