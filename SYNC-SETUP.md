# Настройка облачной синхронизации (Supabase)

Синхронизация нужна, чтобы данные обновлялись между ПК и телефоном. Используем бесплатный Supabase.
Делается один раз, ~5 минут.

## Шаг 1. Создать проект
1. Зайди на **https://supabase.com** → *Start your project* → войди через GitHub или почту.
2. Нажми **New project**. Придумай имя (напр. `jobhelper`), задай **Database Password** (запиши куда-нибудь), выбери регион поближе (Frankfurt/Europe). Нажми **Create**.
3. Подожди 1–2 минуты, пока проект создастся.

## Шаг 2. Создать таблицу
1. Слева открой **SQL Editor** → **New query**.
2. Вставь и выполни (кнопка **Run**):

```sql
create table if not exists backups (
  code       text primary key,
  data       jsonb not null,
  updated_at timestamptz default now(),
  device     text
);

alter table backups enable row level security;

create policy "sync access" on backups
  for all to anon
  using (true) with check (true);
```

> Доступ защищён твоим **кодом синхронизации** (он как пароль). Кто не знает код — не увидит твои данные.
> Если захочешь строже — потом настроим Supabase Auth.

## Шаг 3. Взять URL и ключ
1. Слева **Project Settings** (шестерёнка) → **API**.
2. Скопируй:
   - **Project URL** — вида `https://xxxxxxxx.supabase.co`
   - **anon public** ключ (длинная строка `eyJ...`) — раздел *Project API keys*.

## Шаг 4. Ввести в приложении
На **ПК** и на **телефоне** (в приложении → Настройки → «Синхронизация (облако)»):
1. Вставь **URL проекта** и **ключ (anon public)**.
2. Придумай один и тот же **код синхронизации** для обоих устройств (напр. `misha-zavod-2026`). На обоих устройствах код должен совпадать.
3. Включи галочку **«Синхронизировать автоматически»**.
4. Нажми **Сохранить**, затем **Синхронизировать сейчас**.

Готово. Теперь при изменениях данные будут заливаться в облако и подтягиваться на другом устройстве
(данные сливаются умно — правки с обоих устройств не теряются; для каждого дня берётся более свежая версия).

## Примечание
- Удаление дня на одном устройстве может «вернуться» с другого при слиянии — это осознанное упрощение первой версии. Если понадобится точное удаление между устройствами, добавлю пометки-удаления.
